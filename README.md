# Webserv

## Description

Webserv is a custom HTTP server written in **C++98** as part of the 42 School curriculum.

The goal of this project is to understand how web servers work by implementing HTTP communication, socket programming, request parsing, response generation, and configuration handling from scratch.

The server follows the HTTP/1.1 protocol and uses a non-blocking event-driven architecture to handle multiple clients simultaneously. It supports static website hosting, file uploads, custom error pages, directory listing, CGI execution, and configurable routing inspired by NGINX configuration files.

The server is executed with:

```bash
./webserv [configuration_file]
```

## Features

### HTTP Server

* HTTP/1.1 support
* Non-blocking client/server communication
* Single `poll()` event loop for I/O management
* Multiple simultaneous client connections
* Multiple listening ports
* Browser compatibility
* Accurate HTTP response status codes

### Supported Methods

* GET
* POST
* DELETE

### Configuration

Configuration files allow defining:

* Server ports and interfaces
* Multiple servers
* Error pages
* Maximum client body size
* Routes and locations
* Allowed HTTP methods
* Redirections
* Root directories
* Default index files
* Autoindex
* Upload paths
* CGI execution rules

### Additional Features

* Static file serving
* Custom error pages
* Directory listing
* File uploads
* CGI execution with Python scripts

## Instructions

### Compilation

Compile the project using:

```bash
make
```

Available Makefile commands:

```bash
make        # Compile the project
make clean  # Remove object files
make fclean # Remove object files and executable
make re     # Recompile the project
```

### Execution

Run the server with a configuration file:

```bash
./webserv configs/default_with_error.config
```

Other configuration files can be used depending on the desired test scenario.

### Testing

#### Basic request

```bash
curl -i http://localhost:8080
```

The `-i` option displays the HTTP response headers and status line.

#### Error pages

```bash
curl -i http://localhost:8080/nope.html
```

Expected response:

```
HTTP/1.1 404 Not Found
```

The response body should contain the configured error page.

#### Autoindex

```bash
curl -i http://localhost:8080/files/
```

Expected result:

* HTML directory listing is displayed.
* The server does not return `403 Forbidden` or `404 Not Found`.

#### CGI

```bash
curl -i http://localhost:8080/cgi/hello.py
```

Expected result:

```
HTTP/1.1 200 OK
```

The response should contain the output generated by the CGI script.

Example:

```html
<h1>Hello CGI</h1>
```

#### Upload

```bash
curl -i -F "file=@test.txt" http://localhost:8080/upload
```

## Technical Choices

* Written in C++98 according to the project requirements.
* Uses sockets for network communication.
* Uses `poll()` as the event monitoring mechanism.
* Handles all socket I/O through the event loop.
* Uses a configuration-based architecture inspired by NGINX.
* Uses CGI processes only for script execution.
